# Non Technical Requirements

## Accessibility

The application should be as accessible as possible, the user should be able to use the application with a keyboard only and the application should be usable by screen readers.

## Internationalization

The application should be open to internationalization, but for now the focus will be on english.

## Browser Support

Only modern browsers will be supported, the application will not be supported on older browsers.

## Mobile Support

The application should be responsive in web, however, the goal is to have a native app for mobile devices.

## Loading States

The application should have loading states for all the data loading, for example when the user is browsing recipes, the application should show a loading state while the recipes are loading.

## Error Handling

The application should have error handling for all the data loading, for example when the user is browsing recipes, the application should show an user friendly error state if the recipes fail to load.

## Performance

The application should be performant and should not have any performance issues.

## SEO

The application should be SEO friendly, the application should have a proper title and description.
